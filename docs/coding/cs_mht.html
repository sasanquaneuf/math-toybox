Model

ご提示いただいたコードは、LeetCodeの有名な問題である「Minimum Height Trees (MHT)」を解くためのものですね。木の直径や高さを計算しながら中心に向かっていくロジックがしっかり組まれており、アルゴリズムを深く考察されていることがわかります。

ただ、現在のコードには以下のような課題があり、少し複雑になりすぎています。

  1. 再帰の上限エラーのリスク: calculate_height で再帰 (DFS) を使用しているため、ノード数が数万件のような一直線の木が入力された場合、Pythonの再帰制限 (RecursionError) に引っかかる可能性があります。

  2. 冗長な状態管理: edge_dict を defaultdict(dict) で構築し、後から del で削除していく処理や、高さの最大値・2番目の値を管理する処理が複雑で、バグが入り込みやすい状態です。

  3. データ構造の最適化: 重みなしグラフなので、dict ではなく set や list を使ったほうがメモリ効率も実行速度も上がります。

同じ入出力結果（動作）を保ちつつ、この問題を最も「きれいでPythonic」に解くための標準的なアプローチとして、**「葉の刈り込み（Leaves Trimming）」**という手法があります。外側（葉）から玉ねぎの皮をむくようにノードを削っていき、最後に残った1つか2つのノードが「最小の高さを持つ木の根（木の中心）」になります。

元の意図である「入出力の振る舞い」を完全に保ちつつ、劇的にシンプルにリファクタリングしたコードがこちらです。

Python
from collections import defaultdict
from typing import List

class Solution:
    def findMinHeightTrees(self, n: int, edges: List[List[int]]) -> List[int]:
        # エッジケース: ノードが1つしかない場合
        if n == 1:
            return [0]
        
        # 1. 隣接リストの構築 (set を使用して削除操作を O(1) にする)
        adj = defaultdict(set)
        for u, v in edges:
            adj[u].add(v)
            adj[v].add(u)
        
        # 2. 最初の葉ノード（つながっているエッジが1つだけのノード）をすべて取得
        leaves = [i for i in range(n) if len(adj[i]) == 1]
        
        # 3. ノードの総数が2以下になるまで、外側から葉を削っていく
        remaining_nodes = n
        while remaining_nodes > 2:
            remaining_nodes -= len(leaves)
            new_leaves = []
            
            for leaf in leaves:
                # 葉に接続されている唯一の隣接ノードを取得
                neighbor = adj[leaf].pop()
                # 隣接ノード側からも、この葉への接続を削除
                adj[neighbor].remove(leaf)
                
                # もし隣接ノードが新たに葉（次数1）になったら、次のループの葉として追加
                if len(adj[neighbor]) == 1:
                    new_leaves.append(neighbor)
                    
            leaves = new_leaves
            
        # 最後に残った1〜2個のノードが木の中心（MHTの根）
        return leaves

リファクタリングのポイント

  • ロジックの単純化: 高さを再帰的に計算する代わりに「外側から削る」アプローチに変えたことで、変数の数が激減し、コードの意図がひと目で分かるようになりました。

  • set の活用: defaultdict(set) を使うことで、ノード間のエッジの追加・削除が O(1) の計算量で高速に行えます。

  • メモリと実行速度の改善: 再帰呼び出しによるオーバーヘッドがなくなり、幅優先探索 (BFS) 的に処理を行うため、大規模なデータセットでも安全かつ高速に動作します。
